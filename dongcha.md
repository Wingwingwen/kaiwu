# 🎨 洞察回顾设计规范文档

## 📋 设计概述

洞察回顾系统采用 **3层架构设计**：
1. **选择层** - 5种洞察模式选择
2. **分析层** - AI深度分析展示  
3. **结果层** - 综合报告与可视化

---

## 1. 洞察模式设计 (Review Options)

### 🎯 5种模式配置

| 模式 | 图标 | 主标题 | 副标题 | 主色调 | 理论基础 |
|------|------|--------|--------|--------|----------|
| **relationships** | 🧑‍🤝‍🧑 | 我的人物关系 | 基于社会网络分析 | #FF6B6B | 社会网络分析 |
| **consciousness** | 🎯 | 我的意识层级 | 基于 David Hawkins 意识地图 | #FFD700 | David Hawkins意识地图 |
| **growth** | 🌱 | 我的成长 | 基于David Hawkins意识层级 | #4ECDC4 | 意识层级进化 |
| **attention** | 💡 | 我近期可以注意的 | 基于正念觉察理论 | #FFE66D | 正念觉察理论 |
| **conflicts** | 🔄 | 如何梳理我的内在矛盾 | 基于荣格心理学 | #95E1D3 | 荣格心理学 |

### 📱 界面布局
- **容器**: 圆角卡片设计，白色背景
- **图标**: 40x40px，圆形背景，20%透明度
- **文字**: 标题18px加粗，副标题14px常规
- **间距**: 卡片内边距16px，图标与文字间距12px

## 2. 分析结果展示 (Analysis Results)

### 🧑‍🤝‍🧑 人物关系模式

#### 数据结构
```typescript
{
  "summary": "开篇段落，关于用户的人物关系",
  "people": [
    {
      "name": "人物名称",
      "emoji": "❤️",
      "count": 12,
      "gratitude": "用户感恩他们的具体点"
    }
  ],
  "insight": "一个充满爱的洞察"
}
```

#### 视觉设计
- **总结卡片**: 白色圆角，边框1px #E5E7EB
- **人物卡片**: 横向布局，头像+名称+提及次数
- **头像设计**: 40px圆形，彩色背景(20%透明度)
- **文字层次**: 名称18px加粗，次数12px灰色，感恩内容14px常规

### 🎯 意识层级模式

#### 数据结构
```typescript
{
  "overallLevel": 450,
  "levelName": "理性",
  "distribution": { "low": 20, "mid": 60, "high": 20 },
  "levelBreakdown": {
    "low": [{"phrase": "语句", "level": 100, "levelName": "恐惧"}],
    "mid": [{"phrase": "语句", "level": 310, "levelName": "意愿"}],
    "high": [{"phrase": "语句", "level": 540, "levelName": "喜悦"}]
  },
  "progressSummary": "用户意识进化的总结",
  "encouragement": "来自高维视角的鼓励"
}
```

#### 视觉设计
- **整体层级**: 大号数字显示，彩色文字
- **分布图表**: 三色饼图，红(低维)/蓝(中维)/金(高维)
- **语句分类**: 按维度分组，彩色标签
- **层级标识**: 数字+名称，颜色编码

### 🌱 成长模式

#### 特色功能
- **时间轴**: 显示意识层级变化趋势
- **关键转变**: 高亮重要转折点
- **旅程描述**: 3-4句成长故事

#### 视觉元素
- **当前状态**: 大卡片展示当前层级
- **转变卡片**: 列表展示关键变化
- **鼓励语句**: 温暖色调，手写字体效果

### 💡 注意力模式

#### 核心设计
- **核心洞察**: 每提醒5-10字高亮显示
- **实用建议**: 具体可行的行动指南
- **爱的祝福**: 温暖结尾

#### 布局特点
- **开篇信息**: 简洁2句引入
- **提醒卡片**: emoji + 标题 + 核心洞察 + 详细内容
- **祝福卡片**: 特殊背景色，手写字体

### 🔄 内在矛盾模式

#### 结构框架
- **引入**: 将矛盾视为信使
- **矛盾列表**: 标题+张力描述+整合路径
- **完整智慧**: 关于整合的深刻洞察

#### 视觉处理
- **温和呈现**: 避免对抗性语言
- **整合导向**: 强调和谐与完整
- **智慧升华**: 特殊排版突出

## 3. 综合报告设计 (Comprehensive Report)

### 📊 六边形图表 (Hexagon Chart)

#### 数据结构
```typescript
interface HexagonData {
  love: number;        // 爱 (0-100)
  gratitude: number;   // 感恩 (0-100)  
  joy: number;         // 喜悦 (0-100)
  acceptance: number;  // 接纳 (0-100)
  peace: number;       // 平和 (0-100)
  courage: number;     // 勇气 (0-100)
}
```

#### 视觉规格
- **尺寸**: 200px正六边形
- **颜色**: 渐变色填充，边框2px
- **标签**: 中文+英文双语显示
- **数值**: 中心显示百分比，动态动画

### 🙏 感恩模式分析

#### 内容结构
- **高频感恩**: 柱状图显示top 5
- **感恩对象**: 人物关系网络图
- **感恩成长**: 时间趋势曲线

#### 设计要点
- **温暖色调**: 金色、橙色为主
- **圆润形状**: 避免尖锐角度
- **动态效果**: 加载动画，悬停效果

### 💡 实践建议

#### 布局设计
- **卡片式**: 每个建议独立卡片
- **层次性**: 标题→核心洞察→具体行动
- **可操作性**: 具体、可执行的建议

### 🕊️ 智慧祝福

#### 视觉处理
- **特殊背景**: 渐变背景，柔和阴影
- **手写字体**: 模拟手写效果
- **装饰元素**: 羽毛、光晕等细节

## 4. 交互设计 (Interaction Design)

### 🔄 加载状态
- **思考过程**: 4步骤，每步1.5秒
- **进度指示**: 文字更新+动画效果
- **等待体验**: 教育性内容展示

### ⏱️ 冷却机制
- **时间显示**: 剩余时间格式化显示
- **视觉提示**: 灰色按钮，禁用状态
- **用户反馈**: 震动提醒，友好提示

### 🎯 刷新功能
- **触发条件**: 冷却结束后可用
- **反馈机制**: 震动+加载动画
- **数据更新**: 重新生成分析内容

## 5. 响应式设计 (Responsive Design)

### 📱 移动端优先
- **断点**: 375px (iPhone SE) 为基准
- **字体**: 动态字体大小，14-18px范围
- **触摸**: 最小44px触摸区域

### 🖥️ 平板适配
- **布局**: 卡片网格，2列布局
- **间距**: 增加内边距，20-24px
- **字体**: 适当放大，16-20px

### 🌐 Web端优化
- **最大宽度**: 600px容器限制
- **悬停效果**: 卡片悬停阴影变化
- **键盘导航**: Tab键顺序优化

## 7. 字体与排版 (Typography)

### 📝 字体层级
- **标题1**: 24px, 加粗, 行高32px
- **标题2**: 20px, 加粗, 行高28px  
- **标题3**: 18px, 加粗, 行高24px
- **正文**: 16px, 常规, 行高24px
- **小字**: 14px, 常规, 行高20px
- **注释**: 12px, 常规, 行高16px

### 📐 间距系统
- **基础单位**: 4px
- **卡片内边距**: 16px (4单位)
- **组件间距**: 12px (3单位)
- **段落间距**: 8px (2单位)
- **大间距**: 24px (6单位)

## 8. 动画与过渡 (Animation)

### ✨ 微交互
- **卡片进入**: 0.3s ease-out, 上移20px
- **按钮点击**: 0.1s scale(0.95)
- **加载动画**: 1s 旋转, linear
- **数值变化**: 0.5s ease-in-out, 计数动画

### 🎭 转场效果
- **页面切换**: 0.3s slide, ease-in-out
- **模态框**: 0.2s fade + scale, ease-out
- **图表加载**: 0.8s stagger, 逐个出现

---

## 🎯 设计原则总结

1. **温暖友好**: 避免尖锐角度，使用圆润形状
2. **层次分明**: 清晰的信息层级，重点突出
3. **响应迅速**: 即时反馈，流畅动画
4. **文化适配**: 中英文双语支持，emoji辅助
5. **无障碍**: 高对比度，大触摸区域
6. **性能优先**: 本地缓存，减少网络请求

这套设计规范确保了洞察回顾系统既具有专业性和深度，又保持了温暖、友好的用户体验。
